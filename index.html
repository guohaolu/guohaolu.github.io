search-开发搜索引擎

发表于 2024-09-08

搜索引擎的构成

索引管理器——Index Manager
  1. 索引管理器组件的作用是管理带有索引结构的数据，索引结构是一种用于进行高速检索的数据结构。
  2. 对索引的访问也是通过索引管理器进行的。
  3. 索引管理器通常是将索引作为二级存储上的二进制文件来进行管理的。
  4. 而且，还经常会通过保存经过压缩的索引来达到减少从二级存储加载的数据量，提升检索处理效率的目的。
索引检索器——Index Searcher

索引检索器是利用索引进行全文搜索处理的组件。在大多数情况下，索引检索器都会根据某种标准对与查询相匹配的检索结果排序，并将排在前面的结果返回给应用程序。

索引构建器——Indexer

从文档中构建索引。先通过解析将文本文档分解为单词序列，然后再将该单词序列转换为索引结构。

文档管理器——Document Manager

存储着作为检索对象的文档。结构的映射关系为 文档ID ——> 文档，可以通过现成的 DBMS 实现。

graph LR
    文档编号 -->|映射| 文档

倒排索引的设计

利用索引来进行全文搜索相较于全扫描搜索的效率会高很多，其中索引结构常使用倒排索引。倒排索引会维护单词和文档 ID 的映射关系。

graph LR
    单词 -->|映射| 文档编号-倒排列表
倒排索引的结构

若查询单个单词出现在哪些文档，先通过词典找到对应的倒排列表。

若查询多个单词出现在哪些文档，先通过词典找到对应的倒排列表 L1、L2，取交集得到 L3（= L1 ∩ L2）。

若查询某个短语出现在哪些文档，倒排项中需要维护文档编号和单词出现的偏移量。

分词实现

英文分词实现比较容易，但是中文分词实现比较困难，需要通过一定的策略来实现。

  1. 词素解析，如 Python 的 jieba 库。
  2. N-gram 分隔，将句子分隔成多个 N-gram（Token 词元），这个过程称为：Tokenization。

Java并发集合-并发哈希结构设计

发表于 2024-09-04

并发哈希结构

设计一个支持并发访问的哈希结构（Concurrent Hash Structure）是一项挑战性任务，因为它需要在保证高性能的同时，还要确保数据的一致性和安全性。下面是一些设计并发哈希结构的关键考虑因素和实现策略。

设计目标

  1. 高性能：并发哈希结构应当支持高并发的读写操作，减少锁的竞争，提高整体性能。
  2. 线程安全：在多线程环境下，需要确保数据的一致性和安全性，避免数据竞争和死锁。
  3. 可扩展性：随着数据量的增长，哈希结构应当能够平滑地扩展，支持更多的数据和更高的并发度。
  4. 易用性：设计应当简单易用，易于理解和实现。

设计策略

1. 分段锁（Segment Locking）

  • 原理：将哈希表分成多个段（Segment），每个段有自己的锁。这样，在同一时刻，多个线程可以并发访问不同的段，减少了锁的竞争。
  • 实现：可以使用一个数组来存储多个哈希表，每个哈希表对应一个段，并且每个段有自己的锁。

2. 细粒度锁（Fine-Grained Locking）

  • 原理：在哈希表中，对单个桶（Bucket）进行加锁，而不是整个表。这样可以进一步减少锁的竞争。
  • 实现：每个桶都有自己的锁，当访问或修改该桶时，才对其加锁。

实现细节

  • 锁必须为可重入锁，当线程进行复杂操作的时候，不会死锁。锁同时也需要为读写锁，不会阻塞只读操作。
  • 集合大小在新增元素的时候需要自增，选用原子类型来支撑。
  • 扩容策略，设置一个固定大小的负载因子（平均每个桶存储的元素个数）。
  • add 方法新增元素后会检测集合是否满足扩容条件，若满足则进行扩容操作。扩容操作时不允许集合中存在写操作。

方法介绍：

API 描述
boolean contains(T entry) 是否含有此元素
boolean add(T entry) 新增元素
void acquire(T entry) 获得锁
void release(T entry) 释放锁
void resize() 扩容
boolean policy() 检测是否达到扩容条件

随机过程

随机过程：随机变量的序列，如果我们固定某一个观察时刻t，则事物在时刻t出现的状态是随机的。

伯努利过程：
$$
X = { x_1, x_2, \ldots } = { x_n : n = 1, 2, \ldots ; x_n \in {1, 0} }
$$
从时间和状态空间的维度上，随机过程可以继续被划分：

  1. 离散时间：如果参数 n 取值于离散集，比如整数集Z 或非负整数集 N，则该过程是一个离散时间随机过程。
  2. 连续时间：如果参数n可以取任意实数值，则称其为连续时间随机过程。

对于每个随机变量 **xn**：

  1. 离散状态空间：如果每个随机变量 **xn**只能取有限个或可数无限个值，那么这个随机过程就称为离散状态空间随机过程。例如，伯努利过程中的随机变量只能取0或1两个值。
  2. 连续状态空间：如果每个随机变量 xn 可以取任意实数值或更高维度的值，那么就是连续状态空间随机过程。

马尔可夫链

马尔可夫链（Markov Chain）是一种特殊的随机过程，它在离散时间上进行，并且具有一个重要的特性——马尔可夫性质。

马尔可夫性质指的是，在给定当前状态的情况下，未来的状态仅依赖于当前状态，而不依赖于过去的历史状态。换句话说，一旦当前状态确定了，过去的状态就不再提供额外的信息来预测未来。

  • 状态空间S：可以是有限或无限。

  • 初始状态分布：马尔可夫链开始于某个初始状态的概率分布 π0。

  • 转移概率分布

时间齐次的马尔可夫链：转移概率分布与 t 无关。

转移概率矩阵

如状态空间 S 中仅有三种状态，则转移概率矩阵为 3*3 矩阵，元素定义为由一种状态转为另一种状态的概率。这样只要有了转移概率矩阵，我们就可以从一个过往的概率分布链式拓展下去。
